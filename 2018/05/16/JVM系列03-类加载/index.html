JVM系列03 - 类加载

  • Java
  • 2018-05-16

JVM向程序员屏蔽了操作系统、硬件等不同底层操作，实现了Java语言跨平台的特性。而Class文件屏蔽了不同的语言与JVM的交互，如Java、kotlin、scala等语言都需要编译成字节码才能在JVM中运行，JVM并不关心是何种语言。

什么时候触发类加载？

  • 当程序遇到new关键字，而对应的类没有加载过
  • 读取一个类的静态字段时（但是特别注意如果是被final修饰是不会触发类加载，因为被static final修饰的字段会被认为是一个常量，会放到常量池中）
  • 调用一个类的静态方法时
  • 反射调用类时
  • 初始化类时，如果父类没有加载，触发父类的加载；JDK 8之后接口如果定义的default方法，那么子类初始化之前触发该接口的初始化
  • JVM启动时，触发加载main函数的类

几个特殊的情况：

  • 通过子类引用父类的静态字段，不会导致子类加载
  • 通过数组定义引用类，不会触发此类的初始化
  • 被static final修饰的字段会被认为是一个常量，会放到常量池中，不会触发该类的加载

类加载的流程

加载（Class Loading）

加载阶段主要完成以下三件事情：

  • 通过一个类的全限名来获取此类的二进制字节流
  • 将这个字节流所代表的静态存储结构转换成方法区的运行时数据
  • 在内存中生成一个代表该类的java.lang.Class对象，作为方法区这个类的数据访问入口

双亲委派模式

双亲委派模式的工作流程：如果一个类收到类加载的请求，它首先不会自己尝试去加载这个类，而是把这个请求委派给父类加载器去完成，每一个层次的类加载器都是如此，因此所有的类加载器所有的请求会最终落到顶层的启动类加载器中，只有当父类反馈无法完成这个类的加载请求时，子类才会尝试自己去完成加载。

使用双亲委派模式的目的：让类加载器加载类有一个优先级的顺序，保证程序正确性，从而防止类冲突问题。
