Java并发编程实战

  • 编程语言
  • 2018-04-25

Java并发编程领域的知识非常的多，也非常的杂，但是可以抽象成三个方面：分工、同步/协作和互斥（保证线程安全）三个核心。

分工

类比现实世界，开始一个迭代之前，都需要将任务拆分成很多的小任务，然后分给不同的组员去执行。并发编程中，可以将线程当成是组员，而程序员可以认为是项目经理，通过一些Java的工具将任务分给不同的线程。

Java SDK并发包提供的Executor，fork/join、Future都可以认为是分工思想的实现，还有一些分工的方法，如生产者-消费者模型，Thread-per-message，Work Threrad等。

线程池

线程是较为重量级的资源，创建一个线程需要大概1M左右的内存。线程池通过池化思想和生产者-消费者模式，来提高程序的性能（响应时间、吞吐量），并且降低对系统的压力。Java提供的线程池的构造函数如下：

  • corePoolSize 核心线程池线程数量，当线程池的数量小于corePoolSize时，创建的线程不会被销毁
  • maximumPoolSize 线程池允许的最大线程数量
  • keepAliveTime, unit 当线程数量大于核心线程池数量时，计算消费线程需要的时间
  • workQueue 阻塞队列，生产者-消费者模式，提高线程池吞吐量，一般设置为有界的阻塞队列，防止内存溢出
  • threadFactory 线程工厂，创建线程时用，可以允许用户设置一些具备业务意义的线程名，方便问题定位
  • RejectedExecutionHandler 线程池和队列如果都满了，执行对应的拒绝策略
  • workQueue、threadFactory、handler为空时，会抛出空指针异常；当corePoolSize < 0，keepAliveTime < 0，maximumPoolSize <= 0，maximumPoolSize < corePoolSize时，会抛出非法的参数异常。

CyclicBarrier

CyclicBarrier又叫回环栅栏，通过它可以实现让一组线程等待至某个状态之后再全部同时执行。叫做回环是因为当所有等待线程都被释放以后，CyclicBarrier可以被重用。我们暂且把这个状态就叫做barrier，当调用await()方法之后，线程就处于barrier了。

Synchronized关键字

同步原语，保证一个时间节点只有一个线程能够进入临界区，可以解决原子性，有序性和可见性问题。

Lock

相比Synchronized关键字，Lock相关实现类具备更加丰富的功能，但是和Synchronized相比，最大的不同就是

  • Lock能够响应中断
  • Lock能够超时
  • Lock支持非阻塞的获取锁。

并发容器

java提供的集合框架绝大数都不保证线程安全，所以并发场景下，需要使用并发包相关容器。

CopyOnWriteArrayList

CopyOnWriteArrayList使用copy-on-write的思想来解决并发问题，即新增数据的时候，会拷贝一份数据进行操作，完成后替换掉就得数据结构，适用于读多写少的场景。

ConcurrentHashMap

ConcurrentHashMap操作和HashMap基本相同，但是保证线程安全。原理是采用分段锁的技术，即锁的粒度比较小，不会使用一个全局锁，通过这种方式，可以让多个线程同时操作，key不能为空。

队列

Java 并发包队列是比较复杂的，可以从两个维度来分类。一个维度是阻塞与非阻塞，另一个维度是单端与双端。Java 并发包里阻塞队列都用 Blocking 关键字标识，单端队列使用 Queue 标识，双端队列使用 Deque 标识。

单端非阻塞队列：其实现是 ConcurrentLinkedQueue，双端非阻塞队列：其实现是 ConcurrentLinkedDeque。
